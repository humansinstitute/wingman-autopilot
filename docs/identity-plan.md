# Wingmen Identity Integration Plan

## Goal
Land the Applesauce SDK–based identity system inside Wingmen so users can
authenticate with NIP-07, locally generated keys, or bunker remote signers
while the orchestrator and UI understand each user’s `npub`.

## Work Packages

### 1. Server Layer
- **Deliverables**:
  - `POST /api/auth/session` endpoint issuing a 3-day HttpOnly session cookie.
  - Cookie validation middleware that recovers the authenticated `npub`.
  - Request context plumbing so orchestrator session creation and logging
    include the `npub`.
- **Success Criteria**:
  - Manual API call returns 200 with `{ expiresAt }` and sets cookie.
  - Subsequent authenticated requests expose the same `npub` to downstream
    handlers without re-signing.

### 2. Client/UI Layer
- **Deliverables**:
  - Dashboard identity panel wiring (`wireLocalIdentityPanel`, NIP-07, bunker).
  - Password prompt UX (`uiPrompts.ensurePassword`) and encrypted `nsec`
    caching logic.
  - Visual indicators for active `npub`, session expiry countdown, and logout.
- **Success Criteria**:
  - New keys, NIP-07, and bunker flows each trigger `/api/auth/session` and
    place the UI in an “authenticated” state showing the `npub`.
  - Logout purges encrypted caches and clears the UI state.

### 3. Storage & Orchestrator Layer
- **Deliverables**:
  - Persistence of per-user session metadata (e.g., map `npub` → active agent
    sessions, data directories, audit logs).
  - Tooling to scope filesystem/log access by `npub` where applicable.
  - Reporting hooks so operators can query active identities and history.
- **Success Criteria**:
  - Launching an agent after login tags the session with the user’s `npub`.
  - Data generated by an agent resides in the user’s scoped location.
  - Operators can list sessions grouped by `npub` via existing dashboards or a
    CLI endpoint.

## Integration Order
1. Implement the server package, since both client and orchestrator rely on
   the session cookie contract.
2. Wire the client flows to call the new endpoint and surface the identity
   state.
3. Extend orchestrator storage and reporting once authenticated requests are
   flowing end-to-end.

## Dependencies & Risks
- Bun server must expose fetch-compatible APIs for session issuance.
- Password handling needs careful UX to avoid secret leakage.
- Orchestrator tagging could surface legacy data that lacks `npub` metadata;
  plan for migration or backward-compatible defaults.

## Ready-To-Start Checklist
- Applesauce dependencies installed (`applesauce-signers`, `applesauce-relay`,
  `applesauce-core`).
- Nostr relay list configurable via `NOSTR_RELAYS` env variable.
- Agreement on session cookie naming and security attributes (HttpOnly,
  `SameSite=Lax`, `Secure` in production).
