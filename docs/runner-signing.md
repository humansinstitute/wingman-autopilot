# Runner Signing

Autopilot supports a split signing model for scripts generated by agents:

- the agent runtime does not receive `WINGMAN_PRIV`, `AGENT_NSEC`, API keys, or runner signing tokens
- Wingman keeps the instance key in the server process
- an external runner receives a short-lived `WINGMAN_SIGNING_TOKEN`
- scripts call `bun clis/wingman-sign.ts` when they need a signed NIP-98 token or Nostr event

## Configuration

Set these only on the Wingman host process:

```env
WINGMAN_PRIV=nsec1...
WINGMAN_SIGNING_SECRET=<long random secret>
```

`WINGMAN_SIGNING_SECRET` signs runner capability tokens. Do not inject it into
agent sessions or generated scripts.

## Mint a Runner Token

Mint tokens outside the agent runtime:

```sh
WINGMAN_SIGNING_SECRET=... bun clis/wingman-sign.ts mint \
  --ttl-seconds 600 \
  --session-id session-123 \
  --nip98-host api.example.com \
  --nip98-method POST \
  --nostr-kind 30078
```

Inject only the returned token into the external execution environment:

```sh
export WINGMAN_SIGNING_TOKEN=wmsign_v1...
export WINGMAN_URL=http://127.0.0.1:3600
```

## Script Usage

For NIP-98:

```sh
bun clis/wingman-sign.ts nip98 \
  --url https://api.example.com/jobs \
  --method POST \
  --body-file payload.json \
  --out header
```

For Nostr event signing:

```sh
bun clis/wingman-sign.ts nostr-event \
  --kind 30078 \
  --content-file event.json \
  --tags-json '[["d","config"]]'
```

The CLI is a thin client. It never reads `WINGMAN_PRIV`; it calls the local
Wingman signing API, which verifies the scoped token before signing.
