# Commit Changes

Follow these steps to create a well-structured git commit:

1. Run `git status` to review all changed, added, and deleted files
2. Run `git diff` to understand what changed and why
3. Stage only the files relevant to this logical change — avoid `git add .` unless all changes are related
4. Write a commit message in imperative present tense (e.g. "Add user authentication endpoint")
5. Keep the subject line under 72 characters
6. Do NOT include AI attribution in the commit message
7. If changes span multiple concerns, split them into separate commits
